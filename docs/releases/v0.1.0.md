## Summary

This is the first reproducible portfolio release of SECOP Intelligence. It
turns a bounded official SECOP II cohort into a traceable queue for human
review while preserving a strict privacy boundary and avoiding automated legal
or fraud conclusions.

## Demonstrated capabilities

- API ingestion with stable pagination, completeness checks and safety limits.
- Privacy minimization through explicit field allowlists.
- Deterministic curation, conflict quarantine and content hashing.
- Versioned rules that retain evidence for every finding.
- Typed analytical modeling in DuckDB.
- A responsive Spanish interface for transparent review workflows.
- Portable review artifacts containing CSV data and an integrity manifest.
- A safe synthetic mode suitable for public hosting.

## Accepted full-scale evidence

The local acceptance run used official, privacy-minimized SECOP II records for
Valle del Cauca signed between 2026-01-01 and 2026-07-01.

| Measure | Accepted result |
| --- | ---: |
| Contracts | 51,662 |
| Curated modifications | 523,017 |
| Deterministic findings | 69,794 |
| Contracts with findings | 41,363 |
| Interactive filter response | Under 3 seconds |

Generated source data and the local DuckDB warehouse are intentionally excluded
from Git. The public application creates an explicit synthetic database with
36 demonstration contracts.

The public deployment was externally verified on 2026-07-23. It loaded without
authentication and displayed the required synthetic-data boundary.

## Reproduce the public demo

```bash
uv sync --locked --all-groups
uv run ruff check .
uv run ruff format --check .
uv run pytest
SECOP_DEMO_MODE=1 uv run streamlit run app.py
```

## Release controls

- CI must pass on the exact commit selected for the release.
- The Git working tree must be clean before creating the annotated tag.
- An automated test requires `pyproject.toml` and package `__version__` to
  match `0.1.0`.
- Public demo generation permits only synthetic entities and omits process
  URLs.
- Spanish filters, case selection and ZIP export were visually accepted.
- The annotated `v0.1.0` tag is created only after these controls pass.

## Known limitations

- The public deployment does not include or retrieve the accepted official
  cohort.
- Rules are deterministic review aids and do not establish irregularity,
  corruption, fraud or legal liability.
- The interface is intentionally read-only and has no user-management layer.
- Data refresh orchestration and notifications are outside this release.
